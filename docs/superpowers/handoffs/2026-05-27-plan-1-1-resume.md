The plan was written against Next.js 15.x. **`pnpm create next-app` installed Next.js 16.2.6.** This is acceptable per the spec ("Next.js 15+"), but Next 16 has several breaking changes:

1. **`middleware.ts` was renamed to `proxy.ts`.** Task 7 correctly placed it at `src/proxy.ts` with `export async function proxy(req)`. **For Task 17 (Vercel deploy):** Vercel may or may not auto-detect `proxy.ts`. Verify before declaring success. Cookies-based auth gate works fine via `@supabase/ssr`.

2. Next 16 ships an `AGENTS.md` and `CLAUDE.md` at the project root by default warning agents about breaking changes. Don't be surprised by them.

3. `await cookies()` (Next 15+ async API) is in use throughout `src/lib/supabase/server.ts` and `src/lib/workspace/*.ts` — correct for Next 16.
